Migration Guide

Coming from Bootstrap or Tailwind CSS? This guide maps the classes and patterns you already know to their VelinStyle equivalents.

From Bootstrap to VelinStyle

VelinStyle uses a BEM-inspired naming convention (velin-block__element--modifier) instead of Bootstrap’s flat class names. The table below shows the most common mappings.

Bootstrap VelinStyle Notes
.btn .btn-primary .velin-btn .velin-btn--primary All color variants follow --variant pattern
.container .velin-container Responsive max-width with inline padding
.row .velin-grid CSS Grid instead of flexbox rows
.col-md-6 .velin-col-6 12-column grid, no breakpoint prefix needed — use container queries
.form-control .velin-input Applies to <input>, use .velin-select / .velin-textarea for others
.card .velin-card Sub-elements: __header, __body, __footer
.alert .alert-danger .velin-alert .velin-alert--danger Variants: --success, --warning, --danger, --info
.badge .velin-badge Color variants via --primary, --success, etc.
.nav .velin-nav Includes toggle, brand, and list sub-elements
.modal <velin-modal> Web Component with built-in focus trapping & Escape to close
.d-flex .velin-flex Utility class, combinable with alignment helpers
.text-center .velin-text-center Text alignment utilities
.mt-3 .velin-mt-3 Spacing scale uses logical properties under the hood
.visually-hidden .velin-sr-only Screen-reader-only content

Quick example

A Bootstrap card rewritten for VelinStyle:

<!-- Bootstrap -->
<div class="card">
  <div class="card-body">
    <h5 class="card-title">Title</h5>
    <p class="card-text">Content</p>
    <a href="#" class="btn btn-primary">Go</a>
  </div>
</div>
<!-- VelinStyle -->
<div class="velin-card">
  <div class="velin-card__body">
    <h5 class="velin-card__title">Title</h5>
    <p>Content</p>
    <a href="#" class="velin-btn velin-btn--primary">Go</a>
  </div>
</div>

From Tailwind to VelinStyle

Tailwind is utility-first — you compose designs from dozens of single-purpose classes. VelinStyle takes a component-first approach: semantic component classes handle the common patterns, while a full utility layer is available for one-off tweaks. The result is shorter markup that stays readable.

Tailwind VelinStyle Notes
flex items-center justify-between .velin-flex .velin-items-center .velin-justify-between Utility classes mirror Tailwind when you need them
bg-blue-500 text-white px-4 py-2 rounded .velin-btn .velin-btn--primary One semantic class replaces five utilities
text-sm font-medium .velin-text-sm .velin-font-medium Typography utilities work the same way
p-4 rounded-lg border shadow .velin-card Component class bundles common combinations
space-y-4 .velin-stack Vertical stack with configurable gap
sr-only .velin-sr-only Identical purpose — hides visually, remains for assistive tech

Component classes vs. utility-first

VelinStyle gives you both approaches. Use component classes for established patterns and utilities for custom layouts:

<!-- Tailwind: utility-only -->
<button class="bg-blue-600 hover:bg-blue-700
  text-white font-semibold
  px-6 py-3 rounded-lg
  transition-colors duration-200
  focus:outline-none focus:ring-2
  focus:ring-blue-500 focus:ring-offset-2">
  Save changes
</button>
<!-- VelinStyle: component class -->
<button class="velin-btn velin-btn--primary
  velin-btn--lg">
  Save changes
</button>

<!-- Focus ring, hover state, transitions
     are all built in automatically. -->

When you do need fine-grained control, reach for utilities:

<!-- Mix component & utility classes -->
<div class="velin-card velin-p-6 velin-flex velin-items-center velin-gap-4">
  <img src="avatar.jpg" alt="" class="velin-rounded-full" style="inline-size:3rem">
  <div>
    <p class="velin-font-semibold">Jane Doe</p>
    <p class="velin-text-sm velin-text-muted">Engineer</p>
  </div>
</div>

CLI: mechanical class migration (velinstyle prefix)

The npm CLI can rewrite class="…" and className="…" in .html, .vue, .jsx / .tsx, and a few other extensions: it adds velin- where a matching utility or component exists in the framework source, optionally maps Bootstrap d-* display classes with --bootstrap-display, and loads an explicit JSON map for everything else (e.g. btn-primary → velin-btn--primary).

npx velinstyle prefix ./src
npx velinstyle prefix ./src --write
npx velinstyle prefix ./src --bootstrap-display --write
npx velinstyle prefix ./src --map ./velinstyle-prefix-map.json --write

If velinstyle-prefix-map.json sits in the same directory you pass as the prefix target (or next to a single file you pass), it is loaded automatically. --map <file> merges on top and wins on duplicate keys. Map precedence for each token: explicit JSON, then catalog match (velin- plus the same token when that class exists in VelinStyle), then Bootstrap display aliases.

JSON format: flat "from": "velin-to", or { "classMap": { … } } (when classMap is present, only that object is read). Keys starting with _ or $ are ignored. Example file in the repo: examples/velinstyle-prefix-map.sample.json.

Dry-run is the default; use --write to apply. Attributes containing { or } are skipped. Review grid and modifier names after a bulk pass.

Key Advantages of VelinStyle

Beyond class-name differences, VelinStyle offers structural improvements that Bootstrap and Tailwind don’t provide out of the box.

Built-in Accessibility

WCAG AA compliance is structural, not opt-in. Focus rings, ARIA attributes, skip links, 44×44 px touch targets, and prefers-reduced-motion support are all included by default.

Native Web Components

Complex interactive patterns like modals, tabs, and accordions are encapsulated as custom elements (<velin-modal>, <velin-tabs>) with focus management and keyboard navigation baked in — no jQuery or React required.

OKLCH Color System

Colors are defined in the perceptually uniform OKLCH space, ensuring consistent lightness across hues. Theme switching is a single data attribute change — contrast ratios are guaranteed by the math, not manual checking.

Container Queries

Components respond to their own container width, not the viewport. Drop a card grid into a sidebar or a full-width section and it adapts automatically — no breakpoint classes to manage.

Logical Properties & RTL

Every spacing and layout rule uses CSS logical properties (margin-inline-start instead of margin-left). Set dir="rtl" and the entire UI mirrors automatically.

Zero JavaScript by Default

The CSS layer works without any JS. Web components are opt-in via a single <script> tag. No build step, no tree-shaking configuration, no runtime dependency.
