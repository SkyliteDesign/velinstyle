WordPress

WordPress erwartet Styles und Skripte über wp_enqueue_style / wp_enqueue_script. VelinStyle kannst du als Dateien im Theme oder Plugin ablegen — oder für schnelle Tests per CDN.

1. Zwei übliche Wege

  • CDN (prototyping): unpkg-Links wie in Getting Started — kein npm auf dem Server nötig.
  • Lokal (empfohlen für Live): gebaute velinstyle.min.css und velinstyle-components.iife.js (oder ESM nur, wenn du ein Build-Setup fürs Theme hast) ins Theme-Verzeichnis legen, z. B. unter assets/velinstyle/.

2. Child-Theme: Enqueue in functions.php

add_action('wp_enqueue_scripts', function () {
  $ver = '0.6.1'; // oder filemtime() für Cache-Busting

  wp_enqueue_style(
    'velinstyle',
    get_stylesheet_directory_uri() . '/assets/velinstyle/velinstyle.min.css',
    [],
    $ver
  );

  // IIFE: kein type="module" nötig
  wp_enqueue_script(
    'velinstyle-components',
    get_stylesheet_directory_uri() . '/assets/velinstyle/velinstyle-components.iife.js',
    [],
    $ver,
    true // in footer
  );
});

Optional zweites Stylesheet für ein Theme-Preset (siehe Themes), ebenfalls mit wp_enqueue_style nach dem Haupt-CSS.

3. Block-Editor (Gutenberg)

Der Editor lädt nicht automatisch dein Frontend-CSS. Wenn Blöcke im Editor wie auf der Website aussehen sollen, musst du VelinStyle zusätzlich für den Editor enqueuen (z. B. enqueue_block_editor_assets) oder gezielt Block-Styles bauen. Viele Teams stylen nur das Frontend und akzeptieren Editor-Abweichungen.

4. Icons (Sprite-URL)

Verwende eine absolute URL zum Sprite, damit sie auf allen Permalinks funktioniert:

<?php
$url = get_stylesheet_directory_uri() . '/assets/velinstyle/velin-icons.svg';
?>
<velin-icon name="check" sprite="<?php echo esc_url( $url ); ?>"></velin-icon>

5. Konflikte mit Theme/Page-Builder

Page-Builder und alte Themes bringen oft eigenes CSS. VelinStyle nutzt den Präfix velin- — trotzdem können globale Resets oder !important-Regeln vom Theme Einfluss nehmen. Lade VelinStyle spät in der Kaskade (hoher Priority-Wert bei wp_enqueue_style oder Reihenfolge im functions.php), wenn nötig.

6. Mit VelinStyle arbeiten

Templates (PHP, Block-Templates, FSE-HTML) enthalten dasselbe Markup wie in der Doku: Container, Buttons, Alerts, optional Web Components. Für Formulare und A11y gelten die gleichen Regeln wie außerhalb von WordPress — siehe Forms und A11y.
