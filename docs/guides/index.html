Integration

So bindest du VelinStyle in typische Stacks ein und arbeitest danach mit denselben Klassen, Tokens und optionalen Web Components wie in einer reinen HTML-Seite.

Leitfäden

  • Bestehendes Projekt — Vite, Webpack, statische Seiten, Konflikte mit anderen CSS-Frameworks.
  • Laravel (Blade + Vite) — npm, Layout, @vite, Theme-CSS.
  • WordPress — Child-Theme oder Plugin, Enqueue, Block-Editor, Performance.
  • Vite & React — templates/vite-react-velinstyle und @velinstyle/react.
  • Prompt Scaffolding — velinstyle scaffold (0.8.0).
  • Responsive Layout — velinstyle layout audit|suggest|fix (0.8.0).
  • Performance audit — velinstyle perf audit|suggest|fix (0.9.0).
  • API reference (generated) — velinstyle docs generate (0.9.0).
  • Velin-Meta (AI agents) — velinstyle meta, velin-agent.json (0.9.0).
  • Generated Markdown hub — docs/generated/.
  • JavaScript runtime API — tree-shaking & lazy register (0.9.0).

Gemeinsam für alle: zuerst Getting Started lesen, dann hier den passenden Stack wählen.
