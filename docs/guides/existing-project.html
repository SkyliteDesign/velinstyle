Bestehendes Projekt

VelinStyle ist ein CSS-Bundle plus optionales JS für Web Components. Du integrierst es wie jede andere Design-Bibliothek: einmal global laden, dann Markup mit velin-*-Klassen und Komponenten ergänzen.

1. Voraussetzungen

  • Build-Tool (empfohlen): Vite, Webpack, Parcel, Rspack — oder ohne Build per CDN (siehe Getting Started).
  • Browser: aktuelle Evergreens; für OKLCH und Container Queries keine veralteten WebViews auf Kundenpflicht machen.
  • Viewport: <meta name="viewport" content="width=device-width, initial-scale=1"> in jeder HTML-Ausgabe.

2. Installation mit npm

npm install velinstyle

CSS zentral in deiner Einstiegstylesheet-Datei importieren (einmal, so früh wie möglich nach einem evtl. eigenen Reset):

@import "velinstyle/dist/velinstyle.css";
/* oder nur Produktion/minimiert über deinen Bundler-Pipeline-Schritt */

Web Components (optional) im JS-Einstieg:

import "velinstyle/dist/velinstyle-components.js";
Ohne ES-Module (z. B. ältere Templates): velinstyle-components.iife.js per normalem <script src="…"> einbinden — siehe Getting Started.

3. Reihenfolge und Konflikte

VelinStyle nutzt @layer. Am stabilsten lädt du VelinStyle nach globalem Reset und vor sehr spezifischem Projekt-CSS, das gezielt in eigene Layer gehört oder nur Layout betrifft.

Der Präfix velin- minimiert Kollisionen mit Bootstrap/Tailwind-Klassennamen. Trotzdem: zwei „Komplett-Frameworks“ parallel vermeiden; lieber VelinStyle als Basis und Rest als schmales Custom-CSS.

4. Themes

Zusätzliche Presets liegen unter velinstyle/dist/themes/*.css. Import wie normales CSS oder per zweitem <link>. Steuerung per data-velin-theme am <html> oder an Sektionen — siehe Themes.

5. CLI (optional)

npx velinstyle init
npx velinstyle build
npx velinstyle scan
npx velinstyle prefix ./src
npx velinstyle prefix ./src --bootstrap-display --map ./velinstyle-prefix-map.json --write

prefix (Dry-Run ohne --write): Ersetzt in class/className-Strings Klassen nach dem VelinStyle-Katalog; optional --bootstrap-display für Bootstrap-d-*. Zusätzlich lädt die CLI automatisch velinstyle-prefix-map.json aus dem angegebenen Ordner (oder überschreibt mit --map) — JSON mit beliebigen Quelltoken → Zielklassen (z. B. btn-primary → velin-btn--primary). Beispiel: examples/velinstyle-prefix-map.sample.json. Details: Migration → CLI prefix.

Scanner, Blueprints und Tokens: siehe Security, README und die vollständige Doku auf velinstyle.info.

6. Danach mit VelinStyle arbeiten

  1. Layout mit velin-container, Grid-Utilities und ggf. Container Queries (Doku: Layout).
  2. Komponenten aus der Referenz übernehmen (CSS Components, Web Components).
  3. Farben über Tokens anpassen statt harter Hex-Werte (Tokens).
  4. Icons: <velin-icon> mit sprite-Pfad auf dein gebautes velin-icons.svg (Icons).
