Auto-generated API reference

VelinStyle extracts API tables from source so CLI and component docs stay in sync with the codebase. Output is committed Markdown; CI fails if you forget to regenerate after changes.

Generate locally

npm run docs:generate
# or
npx velinstyle docs generate
npx velinstyle docs generate --scope components
npx velinstyle docs generate --scope tokens --out docs/generated

Scopes: all, components, tokens, utilities, cli, rules, a11y.

What gets generated

SectionSourceMarkdown
Web Components components/velin-*.js components/
Design tokens src/tokens/*.css tokens/
Utilities src/utilities/*.css utilities/
CLI cli/cli-manifest.json cli/commands.md
Scanner rules cli/scanner-rules-data.js rules/scanner.md
A11y CSS src/a11y/*.css a11y/modules.md

Tokens JSON pipeline

Build custom token CSS from JSON (includes fonts, motion, zIndex blocks):

velinstyle tokens validate --input examples/tokens.full.json
velinstyle tokens build --input examples/tokens.full.json -o custom-tokens.css

Example file: examples/tokens.full.json

CI

GitHub Actions runs npm run docs:generate and requires git diff --exit-code docs/generated to be empty. Regenerate and commit after changing components, CSS, or CLI manifest.
