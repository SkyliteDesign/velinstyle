Laravel (Vite + Blade)

Typisches Setup ab Laravel mit Vite: Abhängigkeit installieren, resources/css/app.css und resources/js/app.js anbinden, im Blade-Layout @vite nutzen.

Pfade und Dateinamen können je nach Laravel-Version leicht abweichen — das Prinzip bleibt gleich.

1. Paket installieren

Im Projektroot (dort, wo package.json liegt):

npm install velinstyle

2. CSS einbinden

In resources/css/app.css (oder deiner Haupt-CSS-Datei, die Vite bündelt):

@import "velinstyle/dist/velinstyle.css";

/* Optional: Theme-Preset */
@import "velinstyle/dist/themes/sharp.css";

Alternativ nur Standard-Theme: ersten Import reicht; Themes dann per data-velin-theme am <html> steuern (siehe Themes).

3. Web Components (Vite / ES-Module)

In resources/js/app.js (wird von Vite als Modul gebündelt):

import "velinstyle/dist/velinstyle-components.js";

Blade muss dafür nur das gebündelte Script ausgeben — üblicherweise über dieselbe @vite-Zeile wie für dein App-JS.

4. Blade-Layout

Minimal im <head> (Vite lädt CSS und JS):

<meta charset="utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
@vite(['resources/css/app.css', 'resources/js/app.js'])

Markup in Views nutzt dieselben VelinStyle-Klassen wie in statischen Beispielen, z. B. velin-container, velin-btn, <velin-modal> usw.

5. Icons (Sprite-Pfad)

<velin-icon> braucht einen erreichbaren Pfad zum Sprite. Häufig kopierst du node_modules/velinstyle/dist/velin-icons.svg nach public/ (z. B. public/vendor/velin-icons.svg) und setzt:

<velin-icon name="check" sprite="/vendor/velin-icons.svg"></velin-icon>

Oder du nutzt den CLI-Sprite-Workflow aus der Hauptdoku, wenn du nur ausgewählte Icons shippen willst.

6. Livewire / Inertia

VelinStyle ist „nur“ CSS + Custom Elements. Livewire rendert HTML wie gewohnt — achte darauf, dass nach DOM-Updates von Livewire keine doppelten Event-Listener auf Velin-Komponenten nötig sind (Standard-Verhalten der meisten Velin-Web-Components). Bei Inertia bleibt die Einbindung über das Root-Layout mit Vite identisch.

7. Produktion

npm run build bündelt Imports wie jedes andere Paket auch. Prüfe die gebaute CSS-Reihenfolge, falls du zusätzlich Tailwind o. Ä. nutzt — besser ein klarer Layer-/Import-Plan als zwei vollständige Utility-Frameworks.
